Console Features

The MintMVP Console provides a complete project management dashboard with these modules:

▶ Pipeline

Visual 6-stage overview (Spec → Dev → Testing → Staging → Production → Monitoring) derived from your project data. No manual setup required.

🌐 Environments

Manage Development, Staging, and Production environments with URL, version, deploy status, and health indicators.

💓 Monitoring

System health dashboard with uptime status (up/down/degraded), error and warning counts, and open incidents. Owners and developers can publish a read-only page at /status/<token> for clients (no login).

⚒ Tests (Enhanced)

Extended with E2E test type, plus optional spec and task linking via spec_id and task_id fields.

📑 Full CRUD API

REST endpoints for specs, tasks, tests, incidents, and reports with GET, POST, PATCH, DELETE support.

🐙 GitHub Integration

Connect via GitHub App to auto-sync branches, pull requests, commits, and GitHub Actions runs. Periodic refresh while you use the project plus optional webhooks.

▶ CI / CD

View GitHub Actions workflow runs and GitHub-linked environment signals synced from your repository.

💰 Finance

Track project milestones, payments, and expenses. Overview with budget, paid, pending, and outstanding balance summaries.

Base URL

Production console: https://console.mintmvp.com. Values below are saved in your browser only.

Where to get the project UUID

In the Console, open any project. The address bar looks like:
https://console.mintmvp.com/projects/<project-uuid>/overview
(you might see /tasks, /api, etc. instead of /overview).
Copy the first path segment after /projects/ — that 36-character hyphenated string is your project UUID. Paste it into “Project ID” below.

Resolved base URL

Tip: append ?origin=https://console.mintmvp.com&project=<uuid> to this page to prefill.

Authentication

Every request must send your secret key in the Authorization header:

Authorization: Bearer mint_your_api_key_here

Create keys in the app under Project → API → API Keys. Keys start with mint_ and are tied to one project.

Milestones API

GET /api/v1/projects/<project_id>/milestones lists milestones with nested slices and linked_task_ids. GET .../milestones/<milestone_id> returns one milestone with full detail. POST, PATCH, and DELETE on those paths create, partially update (merge), and delete milestones with validation aligned to the Milestones UI. List and detail responses include meta.phase_descriptions (short text per lifecycle phase key).

Full details in docs/api-milestones.md.

GitHub Integration

Connect your GitHub repository to auto-sync development data into MintMVP Console.

Setup

  1. Create a GitHub App with repository permissions: Contents (read), Pull requests (read), Actions (read), and Metadata (read). Enable Webhooks. Add Deployments (read) if you want GitHub deploy events in CI sync.
  2. Set webhook URL to https://your-domain/api/github/webhook
  3. Add env vars: GITHUB_APP_ID, GITHUB_APP_PRIVATE_KEY, GITHUB_WEBHOOK_SECRET
  4. Install the App on your org and connect via Settings > Integrations > GitHub in the console.

Webhook Endpoint

POST /api/github/webhook — receives GitHub webhook events. Signature is verified with x-hub-signature-256. Events are deduplicated by x-github-delivery.

Public monitoring status

From Project → Monitoring, owners and developers can publish a read-only status page. Visitors open https://<your-console-host>/status/<token> with no account. The server loads project health and incident counts using SUPABASE_SERVICE_ROLE_KEY after validating the URL token; the key never ships to the browser.

Requires the project_monitoring_public table from Supabase migrations. You can stop publishing or rotate the token anytime from the same Monitoring screen.
